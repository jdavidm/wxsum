wxsum

A Stata Command for processing remote sensing rainfall and temperature data.

Overview

The wxsum package processes remote sensing rainfall and temperature data, generating useful summary statistics across user-defined seasons.

Data Requirements

The command can be used with either rainfall or temperature data from any source. The only requirements are that:

  1. The data is "wide," meaning each location is a row and each column is a rainfall or temperature reading from a different day.
  2. The data is measured daily.
  3. The variable names for each column contain yyyymmdd. The variable names can have any prefix but must contain the year, month, and day.

Syntax

wxsum prefix, ini_month(number) fin_month(number) [options]

Options

  • day_month(number) - Starting and ending day of the season month. Default is 1.
  • ini_day(number) / fin_day(number) - Override the starting and ending day of the season month.
  • temperature_data - Specify that the data is temperature data.
  • rain_data - Specify that the data is rainfall data.
  • prefix(string) - Add a custom prefix to all created variables.
  • growbase_low(real) / growbase_high(real) - Bounds for growing degree days calculation.
  • keep(varlist) - Keep created variables and specified existing variables.
  • save(filename) - Save dataset as filename.

Examples

Rainfall Data

Process rainfall data for a season from mid-March to mid-June:

wxsum pic_, ini_month(3) fin_month(6) day_month(15) rain_data keep(id) save("rain_stats.dta")

Temperature Data (Cross-Year Season)

Process temperature data for a season that crosses a calendar year (November to February):

wxsum t_, ini_month(11) fin_month(2) temperature_data growbase_low(10) growbase_high(30) save("temp_stats.dta")
