wxsum

A command for processing temperature and precipitation data.

Overview

The wxsum package processes remote sensing rainfall and temperature data, generating summary statistics across user-defined seasons. Seasons may fall within one calendar year or span two calendar years; cross-year output is labeled with the year the season starts.

Data Requirements

The command can be used with either rainfall or temperature data from any source. The only requirements are that:

  1. The data is "wide," meaning each location is a row and each column is a rainfall or temperature reading from a different day.
  2. The data is measured daily.
  3. The variable names for each daily column use the supplied prefix followed by yyyymmdd, such as rf_19790515.

Syntax

wxsum stubname, type(rain|temp) ini_month(month) fin_month(month) [options]

Options

  • type(rain|temp) - Specify that the data is either rainfall or temperature data.
  • ini_month(month) - Initial month of the season.
  • fin_month(month) - Final month of the season.
  • ini_day(day) / fin_day(day) - Start and end day of the season.
  • lr_years(#) - Number of preceding years for rolling deviations and z-scores. Default is 10.
  • rain_threshold(#) - Rainfall threshold for rainy-day and dry-spell calculations. Default is 1.
  • gdd_lo(#) / gdd_hi(#) - Bounds for capped growing degree day accumulation.
  • kdd_base(#) - Temperature threshold for killing degree days.
  • gdd_bin(#) - Width of fixed-interval seasonal GDD categories. Creates gddcat_YEAR variables.
  • gdd_binlo(#) - Lower endpoint for regular GDD intervals. Default 0.
  • gdd_binhi(#) - Upper endpoint for regular GDD intervals; values at or above are top-coded.
  • tmp_bin(#) - Total number of daily temperature bin count variables to create per season. Creates tmpbinXX_YEAR variables.
  • tmp_binlo(#) / tmp_binhi(#) - Lower and upper bounds for temperature bin construction.
  • shape(wide|long) - Shape of the final output. Default is wide.
  • keep(varlist) - Keep created variables and specified existing variables.
  • save(filename) - Save dataset as filename.

Examples

Rainfall Example

Process rainfall data in the bundled sample dataset:

use rain.dta, clear
wxsum rf_, type(rain) ini_month(05) fin_month(10) ini_day(15) fin_day(15) rain_threshold(1) keep(hhid) save(rainfall_stats.dta)

Temperature Example

Process temperature data for a season that crosses a calendar year (November to February):

use temp.dta, clear
wxsum tmp_, type(temp) ini_month(11) fin_month(02) gdd_lo(8) gdd_hi(32) kdd_base(32) keep(hhid)

GDD Categories Example

Create seasonal GDD categories:

use temp.dta, clear
wxsum tmp_, type(temp) ini_month(04) ini_day(01) fin_month(09) fin_day(15) gdd_lo(18) gdd_hi(30) kdd_base(32) gdd_bin(100) gdd_binlo(50) gdd_binhi(950) keep(hhid)

Daily Temperature Bin Counts Example

use temp.dta, clear
wxsum tmp_, type(temp) ini_month(04) ini_day(01) fin_month(09) fin_day(15) gdd_lo(18) gdd_hi(30) kdd_base(32) tmp_bin(15) tmp_binlo(0) tmp_binhi(39) keep(hhid)
