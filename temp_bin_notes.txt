-Deschene-Greenstone GDD bins
    - User specifies gdd_bin(#) where # is an integer
    - The # determines the width of the bins (say 500 GDD)
    - The command then creates a series of variables for intervals [0,500), [500,100), etc.
    - The command determines the maximum value of GDD in the dataset which is has unique values for each location-season. It then creates bins until the max value is assigned to a bin. So if the max value is 2,750 it would create bins of 500 width from 0 until 3,000 where observations with GDD values between 2,500 and 2,750 get assigned to the last bin.
    - The command creates variables named bin1_year, bin2_year, etc. for each year in the dataset. Labels for those variables specify the bin width, like GDD bin [0,500).
    - Location-season GDD values get assigned to the binx_year that they fall into and so that binx_year = 1 if GDD in that location-season falls in the interval of that bin and 0 otherwise.
    - Optional gdd_binlo(#) where user specifies the lower bound that starts the counting. The default is 0. If the users specifies gdd_bin(500) and gdd_binlo(750) any value below 750, even if it is <250 get put into that first bin. This reduces the number of sparsely populated bins at the bottom.
    - Optional gdd_binhi(#) where the user specifies a max value so that anything above that value get put into the last bin. In that case, if the users specifies gdd_bin(500) and gdd_binhi(3000) any value above 3,000, even if it is >3,500 get put into that last bin. This reduces the number of sparsely populated bins at the top.
    - This way the command stays unit agnostic. The user just needs to ensure they specify bin width, lows, and highs in the same unit that the data is recorded in.
